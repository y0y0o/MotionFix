## 一、FRDM 概述：它做了什么

FRDM 是 InfiniteDance 数据管线中的第三步（最后一步）——**一个专门修复3D人体动作中脚部瑕疵的模型**。

### 1.1 问题背景

从单目视频重建3D舞蹈动作的管线会引入脚部瑕疵：

```
Step 1: GVHMR 单目估计  ->  脚滑率 28.63%, 抖动 31.89
Step 2: IsaacGym 物理仿真 ->  脚滑率 8.87%,  抖动 78.60 (暴增)
Step 3: FRDM 足部修复    ->  脚滑率 5.09%,  抖动 14.33 (最优)
```

物理仿真能消除穿透和浮空，但无法准确建模不同地面的摩擦系数（木地板、水泥地、舞台地毯...），导致脚部从"滑动"变成"高频抖动"——FRDM 就是为修复这种抖动而设计的。

### 1.2 核心设计

| 设计要点 | 说明 |
|----------|------|
| **局部修复** | 只修根关节/膝盖/脚，上半身保持不动 |
| **自监督训练** | 在高质动捕数据上加噪构造"伪瑕疵"样本，无需配对数据 |
| **扩散模型** | 学习从带噪下半身恢复到干净动作的条件分布 |
| **双阶段推理引导** | 早期几何引导（保结构）+ 后期足部接触引导（保物理） |

### 1.3 关键结果

经 FRDM 修复后的脚滑率（5.09%）低于专业光学动捕设备采集的 FineDance（6.22%），证明该方法不仅能修复单目重建的瑕疵，质量甚至可与专业设备持平。

## 二、FRDM 技术细节

### 2.1 动作表示

每帧下肢动作用四个分量描述：

| 符号 | 含义 | 来源 |
|------|------|------|
| j_v | 根关节线速度 | 直接参数 |
| j_r | 关节旋转（6D连续表示） | 直接参数 |
| j_p | 关节位置 | 由 j_v 积分或 j_r 前向运动学推导 |
| b | 脚-地接触二值标签 | 启发式阈值判定 |

位置有两种推导路径，FRDM 利用这种冗余做一致性约束——速度积分 CumSum(j_v) 和旋转前向运动学 FK(j_r) 推导出的位置应该一致。

### 2.2 训练：自监督 + 局部加噪

```
输入: 高质量动捕数据 x_0（如 FineDance）

Step 1: 前向扩散 — 对 x_0 加噪得 x_t
Step 2: 构造混合样本 x_hat_t:
        上半身 <- x_0 对应部分（保持干净）
        下半身（root/knee/foot）<- x_t 对应部分（带噪声）
Step 3: 去噪网络 f_theta(x_hat_t, t) 预测 x_hat_0
Step 4: 与 x_0 计算损失
```

不需要配对瑕疵数据——干净动作加噪本身就模拟了"下半身被破坏"的场景。

### 2.3 损失函数（四项）

| 损失 | 作用 |
|------|------|
| L_recon | 基础重建 MSE，保持与原动作一致 |
| L_Foot | 脚着地时相邻帧位置变化应为0（反脚滑） |
| L_vel-pos | 速度积分位置与直接位置的 L2 一致性 |
| L_epsilon-rp | 旋转 FK 推导位置与直接位置的偏差，epsilon 容差内不惩罚 |

L_epsilon-rp 是最优雅的设计：epsilon 容差区间内不惩罚，允许脚部修复有微调空间；超出 epsilon 才惩罚，防止修复偏离原动作太远。

### 2.4 推理：双阶段引导

| | 早期 (t > t_th) | 后期 (t <= t_th) |
|---|---|---|
| 名称 | 几何引导 Geometric Guidance | 足部接触引导 Foot Contact Guidance |
| 目标 | 保持全局结构一致 | 精修脚部物理正确 |
| 做法 | 混合预测旋转/位置与原始输入 | 当脚着地时用原始速度替换预测速度 |

## 三、复现计划

### 3.1 数据和依赖

| 项目 | 方案 |
|------|------|
| 训练数据 | FineDance（14.6h, 光学动捕, SMPL-X）或 AIST++（5.2h） |
| 测试数据 | 物理仿真后的瑕疵动作（IsaacGym + PHC 对 GVHMR 估计结果做仿真） |
| 动作表示 | SMPL-X，55关节，含手部和面部 |
| 评估指标 | FSR（脚滑率）、Jitter（抖动）、穿透率 |

### 3.2 复现步骤

1. 准备干净数据：下载 FineDance，提取下肢关节的速度/旋转/位置/接触标签
2. 准备瑕疵数据：GVHMR 从视频估计姿态 -> IsaacGym PHC 物理仿真 -> 带抖动的动作
3. 实现训练管线：前向扩散加噪、局部混合样本构造、去噪网络、四项损失函数
4. 实现推理管线：DDIM/DDPM 采样循环、双阶段引导逻辑、阈值 t_th 调参
5. 评估与对比：与原论文结果对比 FSR / Jitter / 穿透率

### 3.3 预期难点

- 物理仿真环境（IsaacGym）搭建和不同地面摩擦系数的设置
- 脚-地接触标签 b 的提取（高度+速度阈值判定）
- 扩散模型采样速度慢，全量数据处理需大量 GPU 时间

## 四、改进方案：用 Transformer 替代扩散模型

### 4.1 动机

| 维度 | 扩散模型（FRDM 原文） | Transformer（我们的方案） |
|------|----------------------|--------------------------|
| 推理速度 | 多步迭代去噪（~50步） | 单步前向推理 |
| 训练复杂度 | 加噪调度、多 timestep 训练 | 标准 seq2seq 训练 |
| 引导机制 | 需两阶段显式引导 | attention mask / cross-attention 注入条件 |
| 可解释性 | 隐式 | attention 权重可视化 |
| 长序列建模 | 受噪声调度限制 | Transformer 天然适合 |
| 灵活性 | 难以交互式编辑 | 可做逐帧条件控制 |

### 4.2 技术路线

将足部修复建模为条件 seq2seq 任务：

```
输入:  瑕疵动作序列 X = [x_1, x_2, ..., x_T]（T帧）
条件:  脚-地接触标签序列 B = [b_1, b_2, ..., b_T]
输出:  修复后动作序列 Y = [y_1, y_2, ..., y_T]
```

架构：Encoder（编码瑕疵动作）-> 条件注入（脚接触标签）-> Decoder（生成修复动作）

### 4.3 关键技术点

**1. 分段掩码策略（替代"只修下半身"）**

通过输出掩码实现：对上半身关节不计算损失，或使用残差预测（Transformer 只预测修正量 Delta，最终输出 = X + Delta，Delta 只在下半身关节上有值）。

**2. FiLM 条件注入**

用 FiLM 层（Feature-wise Linear Modulation）根据脚接触标签 b 调制 Decoder 中间特征。当 b=1（脚着地）时，将速度输出通道增益调制到接近 0，直接用架构实现"脚着地时速度为零"的物理约束。

**3. 时序一致性约束**

- 相对位置编码（RoPE 或 ALiBi）捕捉邻近帧依赖
- 滑动窗口 attention 限制感受野
- 相邻帧输出变化量的 L2 正则作为时序平滑损失

**4. 推理时无需迭代去噪**

一次前向传播完成修复，无需 50+ 步扩散迭代。处理 100 小时数据集时尤为有价值。

### 4.4 与 FRDM 的设计对齐

| FRDM 设计 | Transformer 等价实现 |
|-----------|---------------------|
| 只修下半身 | 输出 mask / 残差预测 |
| 脚着地速度=0 | FiLM 调制 / 显式 L_Foot 损失 |
| 几何引导（保结构） | skip-connection（残差路径） |
| epsilon 容差 | Huber Loss 替代 MSE |
| 自监督训练 | 同：干净数据加噪构造训练对 |

### 4.5 预期优势

1. 推理速度快 1-2 个数量级（单步 vs 50步迭代）
2. 训练更简单（无需噪声调度、timestep embedding）
3. 条件注入更自然（attention/FiLM 直接注入脚触地标签）
4. 可解释性更好（attention 权重可视化）
5. 可扩展性更强（未来可加入音乐节拍、舞种标签等）

### 4.6 潜在风险与应对

| 风险 | 应对 |
|------|------|
| Transformer 生成可能不够平滑 | 加强时序平滑损失 + 滑动窗口 attention |
| 缺乏扩散模型的"多样性"优势 | 修复任务是确定性的（有 GT），恰好适合 Transformer |
| 训练数据不足 | FineDance 14.6h 足够训练小型 Transformer（~10M 参数） |
| 可能过拟合 | Dropout + 数据增强（时间 warping、小幅旋转扰动） |
