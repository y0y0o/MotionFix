## 4. 闪现机制：selective_replace 的 on/off 切换

### 4.1 工作流程

```
对每帧 t:
  if foot_height[t] < ground + 5cm  AND  foot_velocity[t] > 0.03 m/frame:
      output[t] = 0.5 × original[t] + 0.5 × predicted[t]
      # predicted[t] ≈ 原点附近 → output[t] 被拉向原点 ~1.5m
  else:
      output[t] = original[t]  # 不动
```

### 4.2 修正发生的位置——不连续

每个关节只有 16-17% 的帧被修改：

## 5. 为什么 V8 没有这个问题？

| 对比维度 | V8 | V13 |
|---------|-----|-----|
| 训练噪声倍数 | 1x (基准) | **4.3x** |
| full_output 平均误差 | ~0.05m (合理) | **2.2m (崩溃)** |
| 修正幅度 | 厘米级 (<5cm) | **米级 (~1.5m)** |
| 修正后 Jitter | ~9x | **26x** |
| FSR 降低 | -2.9pp | -3.8pp |

V13 的噪声放大 4.3 倍导致模型学习到**退化策略**：
1. 噪声太大，模型无法学到精确的逐帧修正
2. 模型退化为输出一个"平均安全姿态"（所有关节在原点附近）
3. Loss 在这个退化策略上仍然收敛（因为 L1/L_vel 不惩罚恒定输出）
4. `_selective_replace` 在检测到 skating 时应用这个退化输出，造成米级跳跃

## 6. 因果链总结

```
V13 训练噪声过大 (4.3x V8)
  → 模型退化为输出"原点附近 standing pose"
    → full_output 误差高达 2-3 米
      → _selective_replace 只在 skating 帧应用 blend_alpha=0.5
        → 修正帧被拉回原点 ~1.5m，未修正帧留在原位
          → 修正组边界处帧间跳跃 ~1.5m
            → 肉眼可见的脚步闪现
              → Jitter = 0.288 (26x 原始)
```

### 改进方向

1. **降低 V13 训练噪声** 或回到 V8 的噪声水平
2. **在 loss 中加入空间一致性约束**（惩罚预测偏离原始位置过远）
3. **限制最大修正幅度**（clamp 单帧修正 < 0.3m）
4. **连续帧修正一致性**——相邻帧一个被修正、一个不被修正时，强制平滑过渡
5. **引入 R² 评分或 pose validity 约束**——当前 loss 不关心预测的绝对坐标是否正确
