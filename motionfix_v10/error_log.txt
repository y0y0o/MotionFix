⚠️  ISSUE #3 — Upper Body Unchanged (possibly by design, but verify)
----------------------------------------------
The model is trained to only receive lower-body distortions and
output full-body corrections. The test script reshapes (T,22,3) to
(T,66) and feeds it directly — meaning upper body enters as-is.
If the upper body in MoMask output already has artifacts, the model
won't fix them because it was never trained to.

  Step 3 (test):         PASS (runs without crash) but model produces
                         unacceptable jitter on most samples.

Root cause: The training data only adds lower-body distortions and
the model learns to fix them. But MoMask output has DIFFERENT
distortion characteristics than the training artifacts (jitter,
skating, noise, smooth). The model over-corrects MoMask motions.

Recommended fixes:
  1. Increase λ_smooth from 0.3 → 1.0-2.0 to penalize jitter more
  2. Add training augment that simulates MoMask-style artifacts
  3. Or train directly on MoMask outputs as distorted inputs paired
     with GT motions as targets (domain adaptation)
  4. Add a temporal consistency constraint (e.g., velocity L1 on
     upper body to prevent propagation)
