## 第一部分:V18 的逻辑(为什么这样设计)

### 1. 要解决的核心矛盾(来自 V17 的发现)

```
低 FSR  → 接触时脚速度→0 → 平段+硬边界 → 抖动高
低抖动  → 脚速度平滑     → 接触时脚在动 → FSR 高
```

物理修正(damp=0.0)和 V17 都卡在这个对抗上:在**位置空间**直接改脚,要么硬冻结
(边界突变=抖动),要么平滑(接触帧被抹动=FSR 涨)。

### 2. V18 的突破:在「速度空间」工作 + 接触硬 mask

核心洞察:**不要直接改位置,而是预测速度场,然后把接触帧的速度 mask 掉,再积分还原位置。**

```
原始脚 XZ 位置  →  原始速度 v_orig = diff(位置)
                        │
模型预测速度残差 residual(init≈0)
                        │
              v_pred = v_orig + residual
                        │
接触权重 w(soft,接触≈1 空中≈0)
                        │
       v_masked = (1 - w) · v_pred        ← 接触帧速度被乘 0 → 脚不滑
                        │
       pos_new = pos[0] + cumsum(v_masked) ← 积分还原,位置天然连续
```

### 3. 为什么这同时降 FSR 和 Jitter

| | 机制 | 效果 |
|---|------|------|
| **FSR** | `(1-w)` 在接触帧≈0,把脚速度结构性地压到 0 | 保证低,**非学习、不可作弊** |
| **Jitter** | w 是 soft sigmoid,速度在进入接触时**平滑地**降到 0(不是硬切) | 边界连续 → 抖动低 |
| **位置连续** | 积分(cumsum)出来的位置天生连续,不会有跳变 | 无 V17 的位置跳变问题 |

关键:soft 接触权重让速度「在落地前的几帧就平滑减速」——这正是「空中减速」思想,
用速度 mask 解析地实现。模型再在此基础上学习把残留抖动磨掉。

### 4. 模型的角色 = 安全、可控

- 初始化:residual ≈ 0 → V18 退化为「解析 soft 速度 mask」基线
- 训练:模型学 residual,只能让结果**更平滑 / 更贴近原始**
- **FSR 被 (1-w) 焊死,模型动不了它** → 消除了 V8-V17 反复踩的失败模式
- 最坏:residual 没用,抖动 = 解析基线;最好:抖动降到接近原始

### 5. 损失函数

```
L = λ_smooth · ‖acc(pos_new)‖²              # 降抖动(主目标)
  + λ_fidelity · ‖pos_new - pos_orig‖·(1-w)  # 防漂移 + 语义保持(只在空中约束)
```
- smooth:惩罚位置二阶差分(加速度)= 抖动来源
- fidelity:只在空中(1-w 加权)拉回原始,防止积分长期漂移、保持脚不脱离腿;
  接触帧不约束(让脚自由 plant)

### 6. 天然消融(回应 Frederick)

- residual=0(解析 soft 速度 mask) vs 训练后(学习精修)
- 硬 mask vs soft mask
- 物理 damp vs V18
- 4-way:Original / Physics / V17 / V18

### 7. 预期风险

- **漂移**:积分速度可能长期偏离原始 → 脚脱离腿。靠 fidelity(1-w 加权)对抗。
- **骨长**:plant 脚会拉伸踝-脚骨(物理也有此问题,BoneCV 监控)。
- 验收标准:**V18 必须 Pareto 占优物理**(FSR ≤ 物理 且 Jitter < 物理)。

### [执行] 第 1 步:模型搭建

- `models/v18.py`:FootRefiner(1D-CNN,**48,840 参数**)+ refine() 积分 + V18Loss + 推理函数
- 自测通过:residual 初始 = 0(zero-init 末层),退化为解析基线

### [关键发现] 第 2 步:先验证解析基线(residual=0,未训练)

10 held-out 结果:

```
              FSR      Jitter    FootErr   BoneCV
Original     16.3%     0.0140    —         —
Physics      11.9%     0.0319    0.014m    0.031
Analytical    3.7%     0.0123    0.38m     0.056   ← V18 residual=0
```

**✅ 重大突破:速度 mask 打破了 FSR-Jitter 对抗!**
- FSR 16.3%→3.7%(比 SOTA OmniControl/MaskControl 的 5.5% 还低)
- Jitter 0.0140→0.0123(**低于原始**,远低于物理 0.0319)
- 两个指标同时降到原始以下 —— 这是 V8-V17 从未做到的

**❌ 暴露问题:FootErr = 0.38m(脚漂移 38cm)**
- 原因:积分 masked 速度,脚累积偏离身体(日志开头预测的「漂移」风险)
- 物理:每个接触段 plant 在入口位置 p_enter,原始滑到 p_exit,我们的脚停在 p_enter;
  接触结束时缺口 δ = 滑动距离;空中用原始速度摆动,但从「落后」位置出发 → 缺口逐周期累积
- 后果:脚脱离腿,视觉会崩

**→ 这恰好证明学习模型是必需的:**
解析版展示了潜力(FSR/Jitter 双降),但**必须靠学习的 fidelity 项把脚拉回身体**。
模型的工作:空中阶段加速度「追上」身体,闭合每周期缺口 δ,把 FootErr 压下来,
同时保住 FSR/Jitter 的增益。这直接回答 Frederick「学习部分贡献什么」。

### [执行] 第 3 步:训练 FootRefiner

- `training/v18.py`,400 epochs,18.9s,λ_smooth=10, λ_fidelity=8
- 训练 fidelity 0.353→0.050(↓7×),smooth 保持极小(0.00017)
- 表面看模型学会了拉回脚

### [严重问题] 第 4 步:4-way 测试暴露漂移没被修复

```
Metric        Original   Physics   Analytical   V18(learn)
FSR ↓         16.3%      11.9%      3.7%         5.8%
Jitter ↓      0.0140     0.0319     0.0123       0.0132
FootErr ↓     0.0000     0.0140     0.3812       0.3956   ← 漂移 40cm,没修好!
BoneCV ↓      0.0050     0.0310     0.0555       0.0673
```

**好消息:FSR 5.8%(≈SOTA 5.5%)+ Jitter 0.0132(<原始 0.0140)—— 指标层面对抗被打破。**

**❌ 严重问题:FootErr 0.396m,学习模型几乎没改善漂移(vs 解析 0.381m,反而 -4%)。**

**根因(代码审查发现):fidelity 损失是 air-weighted `(1-w)`,接触帧被 mask 掉了。
但漂移恰恰发生在接触帧(脚 plant 在某处、累积偏离身体)。air-weighting 让损失
「看不见」接触帧的漂移 → 训练时 fidelity 降到 0.05,但那只是空中的,接触帧的 40cm
漂移完全没被约束。**

训练 fidelity(0.05,air-only)和测试 FootErr(0.40m,全帧)之间的巨大差距,
正是这个 bug 的证据。

**修复方案:de-skated 锚点目标(per-contact-segment re-anchoring)**
- 不再用 air-weighted 原始位置做 fidelity
- 改为构造「去滑目标」:每个接触段内 = 原始入口位置(plant,跟随身体),空中 = 原始
- fidelity = ‖pos_new - 去滑目标‖,**全帧**约束(不再 mask 接触帧)
- 这样接触帧的漂移会被直接惩罚,模型被迫让 plant 位置跟住身体,FootErr 应降到几 cm
- 去滑目标本身 FootErr 应该很小(只差一个滑动量~几 cm),验证后再训练

### [执行] 第 5 步:de-skated 目标重训(均值锚点,全帧 fidelity)

- 改 deskated_target 用接触段**均值**锚点(比入口锚点 FootErr 更小)
- V18Loss 改为全帧 ‖pos_new - 去滑目标‖(不再 air-weighted)
- 验证去滑目标本身:FSR 3.2%,FootErr 0.12m(合理,只差一个滑动量)
- 重训 400 epoch,训练 fidelity 降到 0.052

### [决定性问题] 第 6 步:漂移仍在,且扯断腿

```
Metric        Original   Physics   V18(learn)
FSR ↓         16.3%      11.9%      5.6%      ← 达 SOTA
Jitter ↓      0.0140     0.0319     0.0135    ← 低于原始
FootErr ↓     0.0000     0.0140     0.386m    ← 仍漂移 38cm
BoneCV ↓      0.0050     0.0310     0.0675
```

训练 fidelity(0.052)和测试 FootErr(0.386m)的巨大差距 = **泛化失败 + 根本限制**。

**量化扯腿(小腿 = 膝4→踝7,原长恒定 0.40m):**
```
动作            小腿原长   V18后均值   最大拉伸
p000021 旋转    0.40m     0.998m      2.05m   ← 小腿被拉长 2 米!
p000818 旋转    0.40m     0.890m      0.92m
p012498 复杂    0.40m     0.498m      0.31m
p002530 走路    0.40m     0.398m      0.07m   ← 走路只拉伸 7cm
p001120 走路    0.40m     0.407m      0.09m
```

**根本结论(本次研究最重要的发现之一):**
> 纯脚部位置修正无法把脚 plant 得太远,否则**扯断小腿**。
> 走路类前进主要在摆动相,plant 偏移小(~7cm),尚可;
> 旋转类脚持续移动,plant 累积偏移巨大(踝漂移 → 小腿拉长 2m),视觉崩坏。
> **这正好解释导师观察到的「rotational motions gains limited」。**

**FSR 指标定义在踝关节(7,8),所以要降 FSR 必须移动踝 → 必然牵动小腿。**

**唯一正确的完整解 = V18 速度场 + IK:**
脚平滑 plant(低 FSR+低 Jitter,已验证)后,用 2-bone IK(髋-膝-踝)
重新摆放膝/髋,让小腿骨长保持恒定 → 腿不断。这也是导师提到的 V9 用 IK 的原因。

### 本步小结(V18 的真实价值与边界)

✅ **速度场 mask 打破了 FSR-Jitter 对抗**(FSR 5.6% + Jitter 0.0135,两个都低于原始)
   —— 这是 V8-V17 从未做到的,是真正的方法贡献
❌ **纯脚位置修正受「扯腿」根本限制**,旋转类尤其严重 —— 需要 IK 补全
→ 下一步明确:V18 速度场 + 2-bone IK = 完整方法

## 第三部分:V18 + 2-bone IK(完整方法)

> 2026-06-25。承接「V18 扯断腿」的发现,搭建 IK 补全,过程与所有踩坑忠实记录。

### 1. IK 的逻辑(为什么这样设计)

V18 速度场把踝关节 plant 住(低 FSR + 低 Jitter),但它**自由移动踝/脚的 XZ**,导致:
- (a) 小腿(膝→踝)被拉长到 2m(扯断腿)
- (b) **脚掌翻转**——因为脚趾关节(10/11)被独立移动,和踝不一致

IK 阶段在保留 V18 plant 目标的同时**修复骨架**:
对每条腿 `髋 → 膝 → 踝 (+ 刚性脚趾)`:
1. 用修正后的踝 XZ 作 IK 目标(Y 保持原始)
2. **把目标 clamp 到腿的可达范围**(|髋-踝| ≤ 大腿+小腿)——这一步**阻止无限拉伸/漂移**
3. 解析 2-bone IK(余弦定理)解膝,用**原始膝**确定弯曲平面(膝按自然方向弯)
4. 脚趾**刚性跟随踝**(toe_new = ankle_new + (toe-ankle)_orig)——骨长+朝向保持,**不翻转**

骨长**逐帧**取自原始运动 → 修正后骨架精确复现源骨长(BoneCV ≈ 原始)。

### 2. [关键发现] clamp 方式:3D vs 水平

- **3D 沿髋→踝方向 clamp**:FSR 9.1% 好,但 ContactAcc 掉到 74.8%——因为沿 3D 线拉回会**抬高脚**,planted 的脚浮空(视觉也差)
- **修复:只 clamp 水平漂移,保持脚高度 Y**:ContactAcc 回到 100%,但 FSR 反升到 21.4%(比原始还差!)

### 3. [决定性诊断] 为什么 V18→IK 让 FSR 变差

逐动作看:FSR 变差的动作(p000818/p000021/p012498/p011997)正是 **V18 小腿拉伸最严重**(1.3-2.85m)的动作。

**根因:FSR 变差源于 V18 的「积分漂移」,不是真实 plant 需求。**
真正的原地旋转,|髋-踝| ≈ 腿长(可达);V18 的 cumsum 把踝越积越远(2.85m)→ IK 必须把它拽回可达 → 被拽的脚沿地面滑 = 比原始更严重的 skating。

**→ 不该把 IK 喂给「积分漂移」的 V18 输出,该喂给「去滑 plant-at-mean 目标」(无积分=无漂移=可达)。**

### 4. [突破] De-skate(plant-at-mean,无积分)+ IK

```
            FSR     Jitter   FootErr  BoneCV  ContactAcc
Original   16.3%   0.0140    0.000   0.0050    100%
De-skate+IK 8.2%   0.0295    0.042   0.0050    100%   ← 扯腿修复!
```
- FSR 8.2%(超 SOTA 5.5% 接近),BoneCV 0.0050(=原始,腿完全不断),FootErr 0.042m(无漂移),ContactAcc 100%
- **唯一弱点:Jitter 0.0295(高于原始)**——plant→空中的尖锐边界跳变

### 5. [σ 扫描] 确定 FSR-Jitter 前沿

对 de-skated 目标做高斯平滑(σ)再 IK:

```
σ     FSR    Jitter   (IK 后)
0     8.2%   0.0295
1.0  10.2%   0.0161
1.5  11.1%   0.0131   ← 拐点:FSR、Jitter 双双低于原始
2.0  12.0%   0.0116
3.0  14.0%   0.0101
```
固定高斯 σ=1.5 已经 FSR 11.1% + Jitter 0.0131,**双双击败原始**,骨架完美。这是解析基线。

### 6. [踩坑 ×3] 学习平滑器为何反复学不动

学习器的目标:自适应地比固定高斯更好(长接触段多平滑、过渡处少平滑防 skating)。

- **坑1:λ_smooth·mean(acc²) + fidelity**:smooth 项卡在 0.003056 不动(零初始化 + acc² 梯度太弱 + fidelity 冻结),残差≈0,学习器退化成恒等
- **坑2:回归高斯目标 ‖out-gauss‖ + hold·w²**:hold(λ=3)把脚冻在 deskated,gauss 项卡在 do-nothing 值;残差仍≈2e-5
- **坑3:回归高斯 + 速度反滑**:残差仍几乎不动。**单独调试发现:回归固定高斯,模型最多只能「复制」它,且 L1 残差是高频抵消,小 CNN 学得极慢(400 ep 仅 0.0041→0.0032),根本到不了 0**

**关键修复:不要回归固定高斯,直接优化真实目标**
```
L = λ_jit·mean(acc²)  +  λ_skate·mean(|v|·w³)  +  λ_anch·‖out-deskated‖
```
单样本验证:jit 0.00095→0.000025(24×↓),漂移仅 0.005m。**模型终于真的在平滑。**
- acc² 直接降 jitter(真实目标,不是模仿某个启发式)
- |v|·w³ 只在**实接触**惩罚脚速度 → 边界可在**空中侧**平滑、接触时刹停(防 FSR)
- anch 防漂移

### 7. [核心科学发现] 学习器与高斯在同一前沿

调 (λ_jit, λ_skate, λ_anch) 得到不同工作点:
```
(300, 3, 1):  FSR 14.3%  Jitter 0.0107   ← 偏平滑
(150,25, 2):  FSR  9.1%  Jitter 0.0260   ← 偏 plant
(250,12, 1):  FSR 12.9%  Jitter 0.0114   ← 交付点
```
六组扫描均**未能 Pareto 占优高斯**——它们落在**同一条 FSR-Jitter 前沿**上。

> **诚实结论:骨架被 IK 约束后,FSR↔Jitter 存在不可消除的权衡;学习器沿着与「调好的高斯」同一条前沿移动,不碾压它,但能任选工作点,且都干净地双指标击败原始。**
> 这与全项目的发现一致:**后训练脚滑修正本质是约束满足问题,微型学习器 ≈ 调好的解析滤波器。**
